# Contributing

Thanks for contributing to Stash.

## Policy

- Follow the repository coding style and keep pull requests focused.
- Include tests or verification steps for behavior changes.
- For security fixes, describe impact and remediation clearly in the PR.
- If you run a modified SaaS or internal deployment, we strongly request that you upstream critical bug fixes and security fixes back to this repository when possible.

## Submitting Changes

1. Fork and create a branch for your change.
2. Make minimal, reviewable commits.
3. Open a pull request with:
    - What changed
    - Why it changed
    - How it was tested

## Reporting Security Issues

If you find a sensitive vulnerability, avoid opening a public issue with exploit details before a fix is available. Contact the maintainers privately first when possible.
